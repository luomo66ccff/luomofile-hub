# Security Policy

## Sensitive material

`.env`、SQLite 数据库、上传内容、对象存储凭据、SMTP 密码和公开链接密钥不得提交到仓库。示例配置只保留空值或本地占位符。

## Safe deployment baseline

- 只在回环地址监听应用端口，并在边缘终止 TLS。
- 默认关闭匿名上传；启用时同时配置尺寸限制、限流和滥用处理。
- 会话密钥、公开链接密钥与内部 API token 必须彼此独立。
- R2/COS 凭据只授予单个 bucket 的最小权限。
- 上传目录和数据库不与 Web 服务器静态根目录重叠。

认证绕过、路径穿越、任意文件读取、签名链接伪造或对象键越权属于高优先级问题。请通过 GitHub 私密安全公告报告，公开 issue 中不要附带真实链接、对象键或凭据。
